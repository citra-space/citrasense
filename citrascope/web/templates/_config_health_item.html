{# Shared Configured/Observed value renderer for a HardwareConfigCheck. Expected Alpine scope: a variable ``chk`` holding one check dict with ``configured_fmt``, ``observed_fmt``, ``status``, ``pct_diff`` and (optional) ``source_samples``. Rendering rules: - No observed value yet (config-only metric or no hardware data): show just the configured string, muted. - Status ok: collapse on match — show one value (observed wins since it may include a binning suffix from the config side) in soft green. - Status warning: show both sides on one line with arrow + pct, yellow. - Sample count (slew rate rolling mean): render ``(n=N)`` when present. #}
