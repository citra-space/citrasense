{# Shared Configured/Observed value renderer for a HardwareConfigCheck. Expected Alpine scope: a variable ``chk`` holding one check dict with ``configured_fmt``, ``observed_fmt``, ``status``, ``pct_diff`` and (optional) ``source_samples``. Rendering rules: - No observed value yet (config-only metric or pending): render the configured string inheriting the parent's color (no explicit style). - Status ok: collapse on match — show ``configured_fmt`` in soft green. We deliberately render ``configured_fmt`` (not observed) because the binning suffix (e.g. ``@ 2×2``) lives on the configured side in ``config_health.py``, so displaying it gives the operator full context even though the underlying numeric values agree. - Status warning: show both sides on one line with arrow + pct, yellow. - Sample count (slew rate rolling mean): render ``(n=N)`` when present. #}
