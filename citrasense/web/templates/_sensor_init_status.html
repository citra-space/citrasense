{# Per-sensor init-state badge + Reconnect button, included in the header of every sensor detail page. Why this lives on the detail page (and not on the monitoring card): monitoring is glanceable, status-only. A reconnect bounces the adapter — it's a destructive operator action. Putting it behind the "Open" hop keeps the monitoring card uncluttered and avoids accidental clicks for the common case where every sensor is healthy. The init_state badge still appears on monitoring, so failures are surfaced there; the action just lives one click deeper. Requires the enclosing scope to expose a ``sensor`` Alpine alias (every detail partial does this via the ``x-data`` wrapper in ``_sensor_detail.html``). #} {# Reconnect button. Always visible on the detail page (unlike the monitoring card, where it lived previously) because the detail page is opt-in — an operator who navigated here is specifically managing this sensor and may want to bounce a connection that looks healthy from the outside but has flaked internally. Disabled while a connect is already in flight. #}
